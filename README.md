## API Endpoint & 404 Troubleshooting

If you see `404` / `Not Found`, the Base URL is usually missing `/v1`.

- OpenAI: `https://api.openai.com/v1`
- DeepSeek: `https://api.deepseek.com/v1`
- SiliconFlow: `https://api.siliconflow.cn/v1`
- OpenRouter: `https://openrouter.ai/api/v1`

AutoGLM uses BigModel's full chat endpoint (compatible with OpenAI format):

- `https://open.bigmodel.cn/api/paas/v4/chat/completions`

Tips:
- Do not add `/v1` twice.
- If the provider URL already contains `/v1`, keep it as-is.
- When unsure, compare with RikkaHub's OpenAI provider behavior (`baseUrl + /models` and `baseUrl + /chat/completions`).
